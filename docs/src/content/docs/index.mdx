> **Experimental** — research prototype, not production-ready.

**nodemode** is an experiment in making `node:fs` and `node:child_process` work on Cloudflare Workers by shimming them with R2 + DO SQLite — the same approach [vinext](https://github.com/cloudflare/vinext) uses to make `next/*` imports work on Workers.

## The problem

Cloudflare Workers have no `fs`, no `child_process`, and no shell. Web frameworks work fine (vinext, OpenNext), but tools that need the filesystem and process spawning cannot run at all:


| Tool | What it needs | Workers alone | With nodemode |
|------|--------------|---------------|---------------|
| **zx** (Google) | Shell, pipes, exit codes | Impossible | Works |
| **simple-git** | .git/ filesystem, refs | Impossible | Works |
| **create-next-app** | mkdir, templates, npm | Impossible | Works |
| **lint-staged** | spawn linters, fs.watch | Impossible | Works |
| **AI coding agents** | fs read/write, exec | Impossible | Works |

## Mapping

| Node.js | Cloudflare Primitive | Status |
|---------|---------------------|--------|
| `fs.readFile` / `writeFile` | R2 `get()` / `put()` + SQLite cache | Working |
| `fs.readdir` / `stat` | DO SQLite directory index | Working |
| `child_process.exec` (builtins) | 24 in-DO command emulators | Working |
| `child_process.spawn` (heavy) | Cloudflare Container via `ctx.container` | Working |
| Pipes `cmd1 \| cmd2` | Built-in pipe support | Working |
| Chains `&&` `\|\|` `;` | Built-in chain support | Working |
| `stdin` / `stdout` / `stderr` | WebSocket + DO SQLite buffer | Working |
| `process.env` | Worker env / Secrets | Working |

## Quick Start

```bash
npx nodemode init
npm install
npx nodemode deploy
```

```ts
import { Workspace, createHandler } from "nodemode";

export { Workspace };
export default { fetch: createHandler() };
```

Or use the client SDK from any JavaScript environment:

```ts
import { NodeMode } from "nodemode/client";

const nm = new NodeMode("https://my-worker.workers.dev", "my-workspace");
await nm.writeFile("index.ts", "console.log('hello');");
const result = await nm.exec("cat index.ts | grep hello");
```

## Module Shims — The vinext Approach for Node.js APIs

Just as [vinext](https://github.com/cloudflare/vinext) shims `next/*` imports so Next.js apps run on Workers, nodemode shims `node:fs` and `node:child_process` so Node.js tools run on Workers.

Configure wrangler to alias Node.js modules to nodemode's shims:

```jsonc
// wrangler.jsonc
{
  "alias": {
    "fs": "nodemode/shims/fs",
    "child_process": "nodemode/shims/child_process"
  }
}
```

Now any library that does `import fs from "node:fs"` or `import { exec } from "node:child_process"` transparently gets nodemode's R2+SQLite implementation — no Container needed, $0 cost, &lt;1ms for cached reads.

```ts
// This library code works unchanged on Workers:
import { readFileSync } from "node:fs";    // → nodemode R2+SQLite
import { exec } from "node:child_process"; // → nodemode ProcessManager
```

| Approach | vinext | nodemode |
|----------|--------|----------|
| Shims | `next/link`, `next/router`, `next/image` | `node:fs`, `node:child_process` |
| Enables | Next.js apps on Workers | Node.js tools on Workers |
| How | Reimplements Next.js API on Vite | Reimplements Node.js API on R2+SQLite+DO |
| Cost | $0 (Worker) | $0 (built-in commands in DO) |
