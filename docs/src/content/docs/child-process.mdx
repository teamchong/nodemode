## Overview

nodemode's `ProcessManager` maps `child_process.exec()` to a three-tier execution model:

1. **Tier 1: Shell builtins** — 24 common commands run directly in the DO ($0, &lt;1ms)
2. **Tier 2: JsRunner** — JS/TS execution in the DO's V8 isolate ($0, ~ms)
3. **Tier 3: Container** — native binaries in real Linux via `ctx.container`

## Tier 1: Built-in Commands

These execute in the DO with zero cost and sub-millisecond latency:

| Command | Description |
|---------|------------|
| `echo` | Print arguments |
| `cat` | Read file contents (supports stdin pipe) |
| `ls` / `ls -l` / `ls -a` | List directory |
| `pwd` | Print working directory |
| `env` | Print environment variables |
| `head -n N` | First N lines (file or stdin) |
| `tail -n N` | Last N lines (file or stdin) |
| `wc` | Word/line/byte count (file or stdin) |
| `grep` | Pattern search with `-r` `-n` `-l` `-i` `-v` `-c` flags |
| `mkdir` / `mkdir -p` | Create directories |
| `rm` / `rm -rf` | Remove files/directories |
| `cp` | Copy files |
| `mv` | Move/rename files |
| `touch` | Create empty file or update mtime |
| `basename` / `dirname` | Path manipulation |
| `which` | Locate command (reports builtins) |
| `whoami` | Current user |
| `date` | Current date/time (ISO 8601) |
| `test` | File/string/numeric tests (`-f`, `-d`, `-e`, `-z`, `-n`, `-s`, `-eq`, `-lt`, etc.) |
| `true` / `false` | Exit 0 / exit 1 |
| `printf` | Formatted output |
| `sleep` | No-op (returns immediately in DO) |

### grep flags

| Flag | Description |
|------|------------|
| `-r` | Recursive search — SQLite index for file discovery, parallel R2 reads |
| `-n` | Show line numbers (`linenum:content` or `file:linenum:content`) |
| `-l` | List only filenames with matches |
| `-i` | Case insensitive |
| `-v` | Invert match |
| `-c` | Count matches |

Recursive grep (`-r`) uses the SQLite file index for O(1) file discovery, then fans out parallel reads from the three-tier cache (zerobuf WASM → SQLite → R2). Unsafe regex patterns (nested quantifiers) automatically fall back to literal string matching.

## Tier 2: JsRunner

Commands like `node script.js`, `./script.js`, and `npx tool` execute JavaScript and TypeScript directly in the DO's V8 isolate:

```bash
# All of these run in-DO via JsRunner, no container needed
node server.js
node -e "console.log(1 + 2)"
node -p "Math.PI"
./my-script.js
npx my-tool
```

### Module system

- **CommonJS**: `require()` with circular dependency handling
- **ESM**: `import/export` auto-converted to CJS at load time
- **TypeScript**: Type annotations stripped (no tsc needed)
- **JSON imports**: `require('./package.json')` works
- **Relative resolution**: `./module`, `../module`, `./module.js`, `./module/index.js`

### Node.js builtins available in JsRunner

| Module | What works |
|--------|-----------|
| `fs` | `readFileSync`, `writeFileSync`, `existsSync`, `mkdirSync`, `readdirSync`, `statSync` — backed by FsEngine |
| `path` | `join`, `resolve`, `dirname`, `basename`, `extname`, `relative`, `isAbsolute`, `sep`, `parse`, `format`, `normalize` |
| `crypto` | `createHash` (md5, sha1, sha256, sha512) |
| `http` | `createServer` (callback fires), `request` (backed by fetch), `get` |
| `https` | Alias for http |
| `net` | `isIP`, `isIPv4`, `isIPv6`, `Socket` (emits honest errors) |
| `worker_threads` | `Worker`, `isMainThread`, `parentPort`, `workerData`, `postMessage` |
| `stream` | `Readable`, `Writable`, `Transform`, `PassThrough`, `pipeline`, `finished` |
| `child_process` | `exec` (calls back into ProcessManager) |

### worker_threads

Workers are real isolated JsRunner instances with separate module caches:

```javascript
// main.js
const { Worker } = require("worker_threads");
const w = new Worker("./worker.js", { workerData: { n: 42 } });
w.on("message", (msg) => console.log(msg)); // { result: 84 }

// worker.js
const { parentPort, workerData } = require("worker_threads");
parentPort.postMessage({ result: workerData.n * 2 });
```

Messages are cloned via structured clone (`JSON.parse(JSON.stringify())`) — no shared memory.

### http.request()

Real HTTP requests backed by `fetch()`:

```javascript
const http = require("http");
const req = http.request({ hostname: "api.example.com", path: "/data" }, (res) => {
  res.on("data", (chunk) => console.log(chunk));
  res.on("end", () => console.log("done"));
});
req.end();
```

## Shell Support

### Pipes

Chain commands with `|` — stdout of each stage feeds stdin of the next:

```bash
cat access.log | grep 200 | head -n 5
echo hello world | wc
cat file.txt | grep ERROR | wc
```

### Chains

Execute commands conditionally:

```bash
# && — stop on failure
mkdir -p dist && echo "created"

# || — run on failure (fallback)
cat missing.txt || echo "not found"

# ; — always run all
echo first; echo second; echo third

# Combined
test -f config.json && cat config.json || echo "no config"
```

### Quoted Strings

Single and double quotes are preserved across pipes and chains:

```bash
echo "hello world"
echo 'single quoted'
grep "multi word pattern" file.txt
```

## Tier 3: Container Execution

Commands not handled by Tier 1 or Tier 2 (npm, git, tsc, python, cargo, etc.) are routed to a Cloudflare Container:

```
Workspace DO receives "npm install"
    → Not a built-in, not JS
    → Start container if not running (ctx.container.start())
    → Wait for container ready (ctx.container.monitor())
    → getTcpPort(8080).fetch("/exec") to container-agent
    → Container executes in real Linux with bash
    → Returns stdout/stderr/exitCode
    → Container sleeps after 10 min inactivity
```

### Container Lifecycle

- **Boot on demand** — container starts on first Tier 3 command
- **R2 FUSE mount** — source files available at `/mnt/workspace`, symlinked into `/workspace`
- **Local build artifacts** — `node_modules`, `dist`, `.npm`, `.next`, `build` on fast local disk
- **Snapshot/restore** — SIGTERM triggers tar+zstd of build dirs to R2, restored on next boot
- **Inactivity timeout** — 10 min idle via `setInactivityTimeout()`, container sleeps
- **Health checks** — DO alarm every 30s pings `/healthz` and reconciles R2 index

### Exit Code 127

If no container is available (local dev, test environment), non-builtin non-JS commands return exit code 127 with a "command not found" message. Builtins and JsRunner work everywhere; containers work in production.

## REST API

### Execute a command

```
POST /workspace/{id}/exec
Content-Type: application/json

{"command": "cat src/index.ts | grep export", "cwd": "/project"}
```

Response:
```json
{
  "exitCode": 0,
  "stdout": "export const hello = 'world';\n",
  "stderr": ""
}
```

### List processes

```
GET /workspace/{id}/process/list
```

Returns metadata only (pid, command, status, exitCode). Use `/process/get` for full output.

### Get process details

```
GET /workspace/{id}/process/get?pid=42
```

Returns full process details including stdout and stderr.

## Process Table

- Stores last 1000 processes (older completed entries pruned automatically)
- Output truncated to 4KB per field in storage (full output returned from exec call)
- Tracks: pid, command, args, status (running/done/error), exit_code, timestamps
