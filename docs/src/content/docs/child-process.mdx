## Overview

nodemode's `ProcessManager` maps `child_process.exec()` to a tiered execution model:

1. **Built-in emulators** — common commands run directly in the DO
2. **Cloudflare Containers** — heavy commands run in real Linux containers

## Built-in Commands

These execute in the DO with zero cost and sub-millisecond latency:

| Command | Description |
|---------|------------|
| `echo` | Print arguments |
| `cat` | Read file contents |
| `ls` / `ls -l` / `ls -a` | List directory |
| `pwd` | Print working directory |
| `env` | Print environment variables |
| `head` / `tail` | First/last N lines |
| `wc` | Word/line/byte count |
| `grep` | Pattern search in files |
| `mkdir` / `mkdir -p` | Create directories |
| `rm` / `rm -rf` | Remove files/directories |
| `cp` | Copy files |
| `mv` | Move/rename files |
| `touch` | Create empty file |
| `basename` / `dirname` | Path manipulation |
| `which` | Locate command |
| `whoami` | Current user |
| `date` | Current date/time |
| `test` | File/string tests (`-f`, `-d`, `-e`, `-z`, `-n`) |
| `true` / `false` | Exit 0 / exit 1 |
| `printf` | Formatted output |

## REST API

### Execute a command

```
POST /workspace/{id}/exec
Content-Type: application/json

{"command": "cat src/index.ts", "cwd": "/project"}
```

Response:
```json
{
  "exitCode": 0,
  "stdout": "console.log('hello');\n",
  "stderr": ""
}
```

### List processes

```
GET /workspace/{id}/process/list
```

### Get process details

```
GET /workspace/{id}/process/get?pid=42
```

## Command Parsing

Commands are parsed with basic shell-like quoting:

```bash
echo "hello world"        → echo ["hello world"]
cat 'file name.txt'       → cat ["file name.txt"]
grep pattern file.txt     → grep ["pattern", "file.txt"]
```

## Container Execution (Planned)

For commands not in the built-in set (npm, node, python, cargo, etc.), nodemode will spawn a Cloudflare Container:

```
Workspace DO receives "npm install"
    → Not a built-in command
    → Spawn Container with Node.js image
    → getTcpPort(8080).fetch() to send command
    → Container executes in real Linux
    → Returns stdout/stderr/exitCode
    → Container sleeps (billed only while active)
```

This mirrors the edgebox architecture where commands are tiered between WASM-native execution and OS-level process spawning.
